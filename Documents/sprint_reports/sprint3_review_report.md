# Sprint 3 Review Summary

## Sprint Goal

The goal of Sprint 3 was to extend the functional prototype with core features, strengthen backend logic, and establish a reliable CI/CD pipeline. This sprint focused on integrating Jenkins for continuous integration and delivery, expanding automated testing with improved code coverage, and preparing the application for a functional review. Additionally, we began creating a local Docker image for the project.

---

## Demo Summary

During the sprint review, the following components were demonstrated:

- Extended functional prototype with core features implemented  
- Integration of Jenkins CI/CD pipeline (build, test, JaCoCo coverage)
- Expanded unit test suite with improved code coverage
- Backend and frontend integration with end-to-end functionality
- Local Docker image creation and testing
- Jenkins pipeline showcasing automated build, test, and report generation processes
- Updated JaCoCo code coverage reports with areas of well-tested code highlighted

---

## What Went Well

- Successfully established a robust CI/CD pipeline using Jenkins, automating build, test, and deployment processes
- Extended unit test suite with comprehensive coverage of new and existing features
- All core features were implemented and demonstrated as functional end-to-end
- Backend and frontend integration was completed and working properly
- Docker image was successfully created and tested locally
- Team demonstrated clear understanding of the full product vision
- Code coverage reports were automatically generated and published by Jenkins pipeline

---

## What Could Be Improved

- Initial Docker implementation could be expanded with more detailed configuration for production use
- Some features could benefit from additional optimization and performance refinement
- Documentation for the CI/CD pipeline could be more detailed for team members unfamiliar with Jenkins
- Test coverage could be expanded to cover additional edge cases and boundary conditions

---

## Next Sprint Focus

In the next sprint, the team aims to:

- Further optimize application performance and address any identified bottlenecks
- Expand Docker configuration for staging and production environments
- Conduct security review and implement necessary security improvements
- Continue expanding test coverage and improve code quality metrics
- Prepare for production-ready deployment
